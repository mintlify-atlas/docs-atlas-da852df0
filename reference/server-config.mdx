The server configuration file (`server.cfg`) controls all aspects of your Nitrox server's behavior. This file is automatically created in your save directory when you first start a server.

## Configuration File Location

The `server.cfg` file is located in your server's save directory:

<CodeGroup>
```bash Windows
%APPDATA%/Nitrox/saves/<ServerName>/server.cfg
```

```bash Linux
~/.config/Nitrox/saves/<ServerName>/server.cfg
```

```bash macOS
~/Library/Application Support/Nitrox/saves/<ServerName>/server.cfg
```
</CodeGroup>

<Note>
Replace `<ServerName>` with your actual server name.
</Note>

## Configuration Format

The configuration file uses a simple key-value format:

```properties
# Comment lines start with #
PropertyName=Value
AnotherProperty=100
```

## Network Settings

### ServerPort

<ParamField path="ServerPort" type="integer" default="11000">
  The port number the server listens on for incoming connections.
  
  **Example:**
  ```properties
  ServerPort=11000
  ```
  
  <Warning>
  If you change this port, players must specify it when connecting. Make sure to forward this port in your router if hosting publicly.
  </Warning>
</ParamField>

### MaxConnections

<ParamField path="MaxConnections" type="integer" default="100">
  Maximum number of players that can connect simultaneously.
  
  **Constraints:** Must be greater than 0
  
  **Example:**
  ```properties
  MaxConnections=20
  ```
</ParamField>

### ServerPassword

<ParamField path="ServerPassword" type="string" default="">
  Password required for players to join the server. Leave empty for no password.
  
  **Example:**
  ```properties
  ServerPassword=MySecurePassword
  # or leave empty for public access
  ServerPassword=
  ```
</ParamField>

### AutoPortForward

<ParamField path="AutoPortForward" type="boolean" default="true">
  If enabled, the server attempts to open the port on your router via UPnP.
  
  **Example:**
  ```properties
  AutoPortForward=true
  ```
  
  <Note>
  Your router must support UPnP for this to work. If it fails, you'll need to manually port forward.
  </Note>
</ParamField>

### LANDiscoveryEnabled

<ParamField path="LANDiscoveryEnabled" type="boolean" default="true">
  Determines whether the server responds to LAN discovery requests.
  
  **Example:**
  ```properties
  LANDiscoveryEnabled=true
  ```
  
  <Note>
  When enabled, your server will appear in the LAN servers list for players on the same network.
  </Note>
</ParamField>

## World Settings

### Seed

<ParamField path="Seed" type="string" default="">
  World generation seed. Leave blank for a random spawn position.
  
  **Example:**
  ```properties
  Seed=MyCustomSeed123
  # or leave empty for random
  Seed=
  ```
</ParamField>

### GameMode

<ParamField path="GameMode" type="enum" default="SURVIVAL">
  Default game mode for the server.
  
  **Possible Values:**
  - `SURVIVAL` - Standard survival mode with hunger, thirst, and oxygen
  - `FREEDOM` - No hunger or thirst requirements
  - `HARDCORE` - Permadeath mode
  - `CREATIVE` - Creative mode with unlimited resources
  
  **Example:**
  ```properties
  GameMode=SURVIVAL
  ```
</ParamField>

### KeepInventoryOnDeath

<ParamField path="KeepInventoryOnDeath" type="boolean" default="false">
  Prevents players from losing items when they die.
  
  **Example:**
  ```properties
  KeepInventoryOnDeath=false
  ```
</ParamField>

### PvPEnabled

<ParamField path="PvPEnabled" type="boolean" default="true">
  Enables or disables player versus player damage and interactions.
  
  **Example:**
  ```properties
  PvPEnabled=true
  ```
</ParamField>

## Save Settings

### SaveInterval

<ParamField path="SaveInterval" type="integer" default="120000">
  Time in milliseconds between automatic world saves.
  
  **Constraints:** Must be at least 1000 (1 second)
  
  **Example:**
  ```properties
  # Save every 2 minutes (default)
  SaveInterval=120000
  
  # Save every 5 minutes
  SaveInterval=300000
  ```
</ParamField>

### DisableAutoSave

<ParamField path="DisableAutoSave" type="boolean" default="false">
  Disables automatic world saving.
  
  **Example:**
  ```properties
  DisableAutoSave=false
  ```
  
  <Warning>
  Disabling autosave may result in data loss if the server crashes. Use the `/save` command manually if disabled.
  </Warning>
</ParamField>

### MaxBackups

<ParamField path="MaxBackups" type="integer" default="10">
  Maximum number of backup saves to keep.
  
  **Constraints:** Must be 0 or greater
  
  **Example:**
  ```properties
  MaxBackups=10
  ```
  
  <Note>
  Setting to 0 disables backups entirely. Older backups are automatically deleted when this limit is reached.
  </Note>
</ParamField>

### DisableAutoBackup

<ParamField path="DisableAutoBackup" type="boolean" default="false">
  Disables automatic backup creation.
  
  **Example:**
  ```properties
  DisableAutoBackup=false
  ```
</ParamField>

### PostSaveCommandPath

<ParamField path="PostSaveCommandPath" type="string" default="">
  Path to a script or executable to run after each successful save.
  
  **Example:**
  ```properties
  # Windows
  PostSaveCommandPath=C:\Scripts\backup-world.bat
  
  # Linux
  PostSaveCommandPath=/home/user/scripts/backup-world.sh
  ```
  
  <Note>
  Useful for custom backup solutions or external monitoring scripts.
  </Note>
</ParamField>

## Performance Settings

### InitialSyncTimeout

<ParamField path="InitialSyncTimeout" type="integer" default="120000">
  Time in milliseconds to wait for initial world sync when a player connects.
  
  **Example:**
  ```properties
  InitialSyncTimeout=120000
  ```
  
  <Note>
  Increase this value if players are timing out during initial connection on slower networks.
  </Note>
</ParamField>

### CreateFullEntityCache

<ParamField path="CreateFullEntityCache" type="boolean" default="false">
  If enabled, caches entities for the entire map on next server start.
  
  **Example:**
  ```properties
  CreateFullEntityCache=false
  ```
  
  <Warning>
  **WARNING!** Enabling this will make server startup significantly longer, but players will experience better performance when exploring new areas.
  </Warning>
</ParamField>

### SafeBuilding

<ParamField path="SafeBuilding" type="boolean" default="true">
  When enabled, rejects build actions detected as desynchronized.
  
  **Example:**
  ```properties
  SafeBuilding=true
  ```
  
  <Note>
  This prevents building glitches but may occasionally reject legitimate builds on high-latency connections.
  </Note>
</ParamField>

## Permission Settings

### AdminPassword

<ParamField path="AdminPassword" type="string" default="randomly generated">
  Password players use with `/login` command to gain admin permissions.
  
  **Example:**
  ```properties
  AdminPassword=MySecureAdminPass123
  ```
  
  <Warning>
  This password is auto-generated on first run. Change it to something secure and keep it private!
  </Warning>
</ParamField>

### DefaultPlayerPerm

<ParamField path="DefaultPlayerPerm" type="enum" default="PLAYER">
  Default permission level for new players.
  
  **Possible Values:**
  - `PLAYER` - Basic player permissions
  - `MODERATOR` - Can kick, mute, and use moderation commands
  - `ADMIN` - Full server control
  
  **Example:**
  ```properties
  DefaultPlayerPerm=PLAYER
  ```
</ParamField>

### LocalhostIsAdmin

<ParamField path="LocalhostIsAdmin" type="boolean" default="true">
  If enabled, players connecting from localhost automatically get admin permissions.
  
  **Example:**
  ```properties
  LocalhostIsAdmin=true
  ```
  
  <Warning>
  Disable this if you're using a proxy server or want stricter permission control.
  </Warning>
</ParamField>

## Player Default Stats

These settings control the starting stats for new players.

### DefaultOxygenValue

<ParamField path="DefaultOxygenValue" type="float" default="45">
  Starting oxygen level for new players.
  
  **Example:**
  ```properties
  DefaultOxygenValue=45
  ```
</ParamField>

### DefaultMaxOxygenValue

<ParamField path="DefaultMaxOxygenValue" type="float" default="45">
  Maximum oxygen capacity for new players.
  
  **Example:**
  ```properties
  DefaultMaxOxygenValue=45
  ```
</ParamField>

### DefaultHealthValue

<ParamField path="DefaultHealthValue" type="float" default="80">
  Starting health for new players.
  
  **Example:**
  ```properties
  DefaultHealthValue=80
  ```
</ParamField>

### DefaultHungerValue

<ParamField path="DefaultHungerValue" type="float" default="50.5">
  Starting food level for new players.
  
  **Example:**
  ```properties
  DefaultHungerValue=50.5
  ```
</ParamField>

### DefaultThirstValue

<ParamField path="DefaultThirstValue" type="float" default="90.5">
  Starting water level for new players.
  
  **Example:**
  ```properties
  DefaultThirstValue=90.5
  ```
</ParamField>

### DefaultInfectionValue

<ParamField path="DefaultInfectionValue" type="float" default="0.1">
  Starting Kharaa infection level for new players.
  
  **Example:**
  ```properties
  DefaultInfectionValue=0.1
  ```
  
  <Note>
  Recommended to keep at 0.1 (the default starting value). Setting to 0 means new players are cured by default.
  </Note>
</ParamField>

## Advanced Settings

### SerializerMode

<ParamField path="SerializerMode" type="enum" default="JSON">
  Format used for saving world data.
  
  **Possible Values:**
  - `JSON` - Human-readable JSON format
  - `PROTOBUF` - Binary format (smaller file size, faster)
  
  **Example:**
  ```properties
  SerializerMode=JSON
  ```
  
  <Warning>
  Changing this after creating a world may require migration. Stick with JSON unless you have a specific reason to change.
  </Warning>
</ParamField>

### DisableConsole

<ParamField path="DisableConsole" type="boolean" default="false">
  Disables the server console.
  
  **Example:**
  ```properties
  DisableConsole=false
  ```
</ParamField>

### IsEmbedded

<ParamField path="IsEmbedded" type="boolean" default="true">
  When true and started in launcher, uses the launcher UI instead of a separate window.
  
  **Example:**
  ```properties
  IsEmbedded=true
  ```
</ParamField>

## Example Configuration

Here's a complete example configuration file:

```properties server.cfg
# Server settings can be changed here

# Network Settings
ServerPort=11000
MaxConnections=20
ServerPassword=
AutoPortForward=true
LANDiscoveryEnabled=true

# World Settings
Seed=
GameMode=SURVIVAL
KeepInventoryOnDeath=false
PvPEnabled=true

# Save Settings
# Measured in milliseconds
SaveInterval=120000
DisableAutoSave=false
MaxBackups=10
DisableAutoBackup=false
PostSaveCommandPath=

# Performance
# Measured in milliseconds
InitialSyncTimeout=120000
# Set to true to Cache entities for the whole map on next run.
# WARNING! Will make server load take longer on the cache run but players will gain a performance boost when entering new areas.
CreateFullEntityCache=false
# When true, will reject any build actions detected as desynced
SafeBuilding=true

# Permissions
AdminPassword=GeneratedPassword123
DefaultPlayerPerm=PLAYER
# If true, players using localhost get admin by default - disable if you're using a proxy server
LocalhostIsAdmin=true

# Default player stats below here
DefaultOxygenValue=45
DefaultMaxOxygenValue=45
DefaultHealthValue=80
DefaultHungerValue=50.5
DefaultThirstValue=90.5
# Recommended to keep at 0.1f which is the default starting value. If set to 0 then new players are cured by default.
DefaultInfectionValue=0.1

# Advanced
# Possible values: JSON, PROTOBUF
SerializerMode=JSON
DisableConsole=false
# When true and started in launcher, will use launcher UI as opposed to external window
IsEmbedded=true
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Configuration not loading" icon="triangle-exclamation">
    **Symptoms:** Settings don't apply, server uses defaults
    
    **Solutions:**
    - Check that the file is named exactly `server.cfg`
    - Verify the file is in the correct save directory
    - Check for syntax errors (use `Property=Value` format)
    - Look for warnings in the server log
  </Accordion>
  
  <Accordion title="Invalid value warnings" icon="circle-exclamation">
    **Symptoms:** Log shows "invalid value" warnings
    
    **Solutions:**
    - Check that numeric values are within valid ranges
    - Verify boolean values are `true` or `false` (lowercase)
    - Ensure enum values match exactly (case-sensitive)
  </Accordion>
  
  <Accordion title="Port forwarding not working" icon="network-wired">
    **Symptoms:** Players can't connect from outside your network
    
    **Solutions:**
    - Set `AutoPortForward=true`
    - Check if your router supports UPnP
    - Manually forward the port in your router settings
    - Verify firewall allows the port
  </Accordion>
</AccordionGroup>
